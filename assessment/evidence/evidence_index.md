# Evidence Index

This index describes the evidence categories used to score CSF outcomes.

| Evidence Type | Description | Typical Source | Relevant Functions |
|---|---|---|---|
| Policy documents | Approved policies, standards, and procedures | Confluence/SharePoint policy library | GV, PR, RS, RC |
| Governance records | Steering committee minutes, risk reviews, decisions | Meeting minutes, risk committee packs | GV, ID |
| Asset and config exports | Inventory snapshots, CMDB exports, cloud config baselines | Endpoint mgmt, cloud portal exports | ID, PR, DE |
| Security logs and alerts | SIEM events, EDR alerts, authentication logs | SIEM, EDR, Entra ID logs | DE, RS |
| Ticketing and workflow records | Incident tickets, vulnerability remediation tickets, exception approvals | ITSM platform | ID, PR, DE, RS |
| Backup and recovery reports | Backup job summaries, restore tests, failure reports | Backup console and change records | PR, RC |
| Training and awareness metrics | Completion rates, phishing simulation outcomes | LMS/phishing platform reports | PR |
| Contract and supplier records | Security questionnaires, due diligence notes, contract clauses | Procurement/legal documentation | GV, ID |

## Evidence quality checks
- Verify evidence date and scope coverage.
- Confirm control owner and accountability.
- Cross-check policy intent against operational records.
- Prefer direct system-generated artifacts over narrative-only claims.
